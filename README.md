# 📅 WpfSportsCalendar

**WpfSportsCalendar** — це сучасний desktop-додаток на платформі WPF (.NET), що є персональним спортивним календарем футбольного вболівальника. Додаток дозволяє відстежувати результати минулих матчів та розклад майбутніх ігор улюблених команд у реальному часі.

---

## 🚀 Основний функціонал

*   Авторизація та сесії: Безпечна реєстрація користувачів з хешуванням паролів (BCrypt) та збереженням довгострокових сесій.
*   Розумний пошук команд: Інтеграція із зовнішнім API-FOOTBALL, що дозволяє знаходити клуби та збірні по всьому світу. Збірні автоматично пріоритезуються у пошуковій видачі.
*   Гнучка система підписок: Можливість підписуватись на улюблені команди без роздування локальної бази даних (зберігаються лише зовнішні ID).
*   Персональний Дашборд: На головному екрані автоматично формується актуальний графік: 1 останній завершений матч та 2 найближчі майбутні матчі для всіх підписок користувача.

---

## 🛠 Технологічний стек

*   **Платформа:** .NET 10 / WPF (C#)
*   **Дизайн / Інтерфейс:** Fluent Theme (сучасний стиль Windows 11)
*   **База даних:** MS SQL Server (через Docker-контейнер)
*   **ORM:** Entity Framework Core (Fluent API, Міграції)
*   **Зовнішнє API:** [API-FOOTBALL (api-sports.io)](https://api-sports.io/)
*   **Шифрування:** BCrypt.Net

---

## 📐 Архітектура бази даних

Локальна база даних спроектована так, щоб бути максимально компактною. Дані про підписки пов’язуються з користувачами за допомогою зовнішніх унікальних ідентифікаторів команд (External IDs), що надаються API.

*   `Users` — дані облікових записів користувачів.
*   `UserSessions` — GUID-токенів для функції «Запам'ятати мене».
*   `Subscriptions` — зв'язувальна таблиця підписок (`UserId` ↔ `ClubId`).

## 📊 Відповідність технічним вимогам курсової роботи

Проєкт успішно реалізує ключові вимоги технічного завдання, охоплюючи просунуті теми екосистеми .NET:

### 1. Технології та Архітектура
*   **Платформа:** Desktop-додаток на базі **WPF** (.NET 10).
*   **Розділення на шари:** Логіка чітко розмежована на контекст даних (`Database`), мережеві сервіси (`Services`) та інтерфейс користувача (`Windows`).
*   **Системне програмування (Асинхронність):** Усі мережеві запити до зовнішнього API та важкі операції з БД виконуються асинхронно з використанням патерну **`async / await`**, що запобігає блокуванню та зависанню UI.
*   **Мережеве програмування:** Інтеграція з зовнішнім REST API здійснена через сучасний асинхронний **`HttpClient`** з використанням розширення `GetFromJsonAsync`.

### 2. База даних та ORM
*   **Технологія:** Реляційна БД **MS SQL Server** під керуванням **Entity Framework Core**.
*   **Зв'язки та відносини:** Реалізовано класичне відношення **Many-to-Many** (Багато-до-Багатьох) між користувачами та зовнішніми клубами через проміжну таблицю підписок `Subscriptions`.
*   **Операції з даними:** Реалізовано асинхронний доступ до БД через контекст, а також логіку **пошуку, фільтрації та розумного сортування** (LINQ) отриманих даних.

### 3. Функціональні модулі
*   **Система користувачів (Identity):** Повноцінний модуль реєстрації, автентифікації та авторизації.
*   **Безпека:** Паролі користувачів хешуються "на льоту" за допомогою криптографічного алгоритму **BCrypt**.
*   **Сесії ("Запам'ятати мене"):** Реалізовано механізм довгострокових сесій із генерацією унікальних `Guid` токенів[cite: 1].
*   **Валідація (Data Annotations):** Вхідні дані перевіряються на коректність на рівні моделей[cite: 1].
